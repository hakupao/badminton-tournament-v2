## 可优化项

### UX / 交互
1. **赛事状态可视化**：除了文字标签，考虑用进度条或步骤条显示 draft→active→finished
2. **赛程矩阵 - 自动定位当前轮次**：打开页面时自动展开/滚动到第一个未完成的轮次
3. **记分界面返回**：记分完成后自动跳转回比赛详情或赛程页
4. **比分录入批量模式**：管理员一次性录入多场比分，减少页面切换
5. **赛程列表视图**：手机端列表视图也可以考虑按「我的比赛优先」排序

### 技术 / 性能
1. **数据刷新策略**：当前各页面独立 fetch，考虑 SWR 或 React Query 统一缓存
2. **API 响应优化**：schedule API 返回完整 matches 数组，数据量大时可分页
3. **离线兼容**：比赛日网络可能不稳定，记分界面考虑 Service Worker 或 IndexedDB 暂存
4. **赛事模拟器 FK 错误**：PUT `/api/tournaments/[id]` 删除 players 时触发 FOREIGN KEY 约束失败，需要先删除 matches → match_games → referee_records 等依赖数据

### 安全
1. **恢复 API 鉴权**：删除所有 `// TODO: Re-enable auth` 注释，添加 `requireAdmin` 中间件
2. **运动员权限控制**：记分接口应验证操作者身份
3. **CSRF 防护**：当前 POST/PUT/DELETE 无 CSRF token
