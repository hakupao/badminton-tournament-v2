## 已知 Bug 和缺口（下一 Session 待处理）

### 🔴 P0 - 影响核心使用流程

#### Bug 1：比分页面硬编码 tournament ID
- **文件**：`src/app/match/[id]/page.tsx` 第 43 行
- **文件**：`src/app/match/[id]/scoring/page.tsx` 第 62 行
- **问题**：两处均写死 `fetch('/api/tournaments/1/schedule')` 查找比赛数据
- **影响**：多赛事场景下，非 ID=1 的赛事比分页/记分页会报「比赛不存在」
- **方案**：新增 `GET /api/matches/[id]` 单场详情接口，或在 schedule API 返回时携带 tournamentId

#### 缺口 2：管理后台「赛程管理」页面不存在（404）
- **链接来源**：`/admin` 首页导航卡片「赛程管理」→ `/admin/schedule`
- **需求**：显示已生成赛程、提供「生成赛程」按钮、支持应急调整
- **当前**：点击直接 404，生成赛程功能隐藏在模拟器里不易发现

#### 缺口 3：管理后台「比分录入」页面不存在（404）
- **链接来源**：`/admin` 首页导航卡片「比分录入」→ `/admin/scoring`
- **需求**：管理员赛后补录，支持逐场选择比赛+输入各局比分+记录裁判
- **当前**：点击直接 404

#### 缺口 4：运动员账号与代号绑定 UI 缺失
- **需求流程**：线下抽签后，管理员将注册账号（如用户名 "zhangsan"）绑定到代号（🐱猫-2）
- **数据库**：`users.player_id` 字段已存在，`/api/auth/me` 也返回 `playerId`
- **缺失**：`/admin/players` 页面只能录入姓名，**没有「绑定账号」的 UI 和接口**

#### 缺口 5：赛事状态管理 UI 缺失
- **需求**：draft（筹备）→ active（进行中）→ finished（已结束）
- **API**：`PUT /api/tournaments/[id]` 已支持修改 `status` 字段
- **缺失**：管理后台无「激活赛事」/「结束赛事」按钮，无法在前端切换状态

### 🟡 P1 - 影响安全/体验
