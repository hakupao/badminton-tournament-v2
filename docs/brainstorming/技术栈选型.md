# 技术栈选型讨论

## 待确认的关键决策

### 前端
- **方案 A**: React + Vite（生态成熟，组件库丰富）
- **方案 B**: Vue 3 + Vite（上手快，中文社区友好）
- **方案 C**: Next.js（全栈框架，SSR/SSG 支持）
- UI 组件库偏好？（Ant Design / Tailwind CSS / shadcn/ui 等）

### 后端
- **方案 A**: Node.js (Express/Fastify)
- **方案 B**: Python (FastAPI/Django)
- **方案 C**: 全栈框架自带后端（Next.js API Routes / Nuxt Server）
- **方案 D**: BaaS（Supabase / Firebase，减少后端开发量）

### 数据库
- SQLite（简单轻量，适合小规模）
- PostgreSQL（功能强大，Supabase 自带）
- MongoDB（灵活 schema）

### 部署
- Vercel / Netlify（前端为主的方案）
- Railway / Render（全栈部署）
- 自己的服务器？
- 纯前端方案（数据存本地/浏览器）？
