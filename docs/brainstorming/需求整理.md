## 三、用户体系（三级，极简注册）

```
┌─────────────────────────────────────────────────────┐
│  管理员（需登录）                                      │
│  • 注册方式：用户名 + 密码                              │
│  • 权限：一切编辑操作                                   │
├─────────────────────────────────────────────────────┤
│  运动员（需登录）                                      │
│  • 注册方式：用户名 + 密码（极简，无需邮箱/手机验证）      │
│  • 抽签后由管理员将运动员账号绑定到具体代号（如 🐱猫-2）  │
│  • 权限：查看自己的赛程/比分/统计 + 担任裁判时记分        │
├─────────────────────────────────────────────────────┤
│  普通用户（无需登录）                                   │
│  • 直接访问，公开查看赛程/比分/排名/统计                 │
└─────────────────────────────────────────────────────┘
```

**运动员绑定流程：**
1. 管理员在系统中创建赛事，设定代号（如 🐱猫-1、🐱猫-2...）
2. 线下抽签，确定每个人对应的代号
3. 运动员自行注册账号（用户名+密码，极简）
4. 管理员在后台将运动员账号绑定到对应代号

**运动员登录后的额外能力（相比普通用户）：**
- 看到「我的比赛」视角（高亮/筛选自己参与的比赛）
- 担任裁判时，可以在记分界面标记「我是裁判/边裁」
- 查看自己的个人统计数据

## 四、核心概念

### 4.1 代号系统（动物系）

模拟阶段还没有具体的人名，使用可爱动物代号标识每个位置：

```
小组命名：🐱猫队、🐶狗队、🐰兔队、🐼熊猫队、🐹仓鼠队、🦝浣熊队、🐨考拉队、🦔刺猬队...
位置代号：队名 + 位号
  🐱猫-1 (男·主力)    🐶狗-1 (男·主力)
  🐱猫-2 (男)         🐶狗-2 (男)
  🐱猫-3 (男)         🐶狗-3 (男)
  🐱猫-4 (女·主力)    🐶狗-4 (女·主力)
  🐱猫-5 (女)         🐶狗-5 (女)
```

**使用场景：**
- 模拟阶段：赛程矩阵显示「🐱猫-1+🐱猫-2 vs 🐶狗-1+🐶狗-2」
- 抽签后：代号绑定真人，显示变为「🐱猫-1 张三 + 🐱猫-2 李四 vs ...」
- 代号贯穿整个赛事，方便快速识别

### 4.2 轮次制赛程

```
传统方式：每场比赛独立安排时间
本系统：  按「轮次」组织，同一轮的所有场地同时开始

┌──────────┬────────────────┬────────────────┬────────────────┐
│          │    场地 1       │    场地 2       │    场地 3       │
├──────────┼────────────────┼────────────────┼────────────────┤
│ 第 1 轮  │ 🐱 vs 🐶      │ 🐱 vs 🐶      │ 🐱 vs 🐶      │
│ 9:00     │ 男双 1+2vs1+2  │ 男双 2+3vs2+3  │ 女双 4+5vs4+5  │
├──────────┼────────────────┼────────────────┼────────────────┤
│ 第 2 轮  │ 🐱 vs 🐶      │ 🐱 vs 🐶      │ 🐰 vs 🐼      │
│ 9:20     │ 混双 1+4vs1+4  │ 混双 3+5vs3+5  │ 男双 1+2vs1+2  │
├──────────┼────────────────┼────────────────┼────────────────┤
│ 第 3 轮  │ 🐰 vs 🐼      │ 🐰 vs 🐼      │ 🐰 vs 🐼      │
│ 9:40     │ 男双 2+3vs2+3  │ 女双 4+5vs4+5  │ 混双 1+4vs1+4  │
├──────────┤                │                │                │
│ ...      │                │                │                │
└──────────┴────────────────┴────────────────┴────────────────┘
```

**规则：**
- 每轮最多同时进行 = 场地数 的比赛
- 同一轮所有场地同时开始
- 等最后一个场地结束 → 全部下场 → 下一轮全部上场
- 每轮时间 = 单场预计时间（默认 20 分钟）
- 总时间 = 轮次数 × 每轮时间

**编排约束：**
- 硬约束：同一人不能在同一轮出现在多个场地
- 软约束：同一人尽量不连续上场
- 软约束：同一人尽量不连续轮空

## 五、功能模块

### 5.1 赛事模拟器（赛前核心工具）

场景：拿到 3 片场地 9:00-19:00，报名人数和赛制都在变化，需反复调参找最优方案。

**可调参数：**
| 参数 | 默认值 |
|------|--------|
| 小组数 n | — |
| 每组男生数 | 3 |
| 每组女生数 | 2 |
| 场地数 | 3 |
| 赛事时间段 | 9:00 - 19:00 |
| 计分方式 | 一局21分 / 一局30分 / 三局两胜21分 / 三局两胜15分 |
| 计分作用域 | 全赛事统一（可选按赛种不同） |
| 每轮预计时间 | 20 分钟 |
| 比赛模板 | 管理员自定义位置搭配 |

**实时输出：**
1. **总览**: 总场次、总轮次、预计总时长、是否超出可用时间
2. **赛程矩阵**: 时间段 × 场地 的课程表视图
3. **排布质量报告**:
   - 每位运动员一行：总上场数、最大连续上场、最大连续轮空
   - 异常标红（如连续 ≥3 轮上场 或 连续 ≥3 轮轮空）

### 5.2 人员管理
- 录入：姓名 + 性别
- 管理员控制人数，保证整除

### 5.3 比赛模板（管理员自定义）
- 定义位置数量和性别（编号越小 = 实力越强）
- 定义两队交锋的比赛列表
- 默认（3男2女=5人组）：
  ```
  男双：1+2 vs 1+2
  男双：2+3 vs 2+3
  女双：4+5 vs 4+5
  混双：1+4 vs 1+4
  混双：3+5 vs 3+5
  → 每次交锋 5 场
  ```

### 5.4 分组
- 系统随机 或 管理员手动录入（线下抽签结果）

### 5.5 赛程自动编排
- 将所有比赛分配到 轮次 × 场地
- 遵守硬约束 + 尽量满足软约束
- 输出排布质量报告
- 支持赛中应急调整

### 5.6 比赛记分（两种方式并存）

**方式一：管理员赛后录入**
- 录入每局比分（如 21:15, 18:21, 21:19）
- 同时记录裁判/边裁

**方式二：裁判手机实时记分**
- 在赛程矩阵中点击当前轮次的比赛，进入记分界面
- 运动员主动选择「我是裁判」/「我是边裁」
- 极简界面：两个大按钮 +1，每得分点一下
- 达到制胜分 → 自动结束（三局两胜则自动进入下一局）
- 比分实时保存
- 记分者自动计入裁判志愿

### 5.7 积分与排名

**团体交锋胜负：**
- 比赢的场数，场数多的一方赢
- 奇数场→一定有胜负；偶数场→可能平局

**积分规则：**
- 赢 3 分 / 平 1 分 / 负 0 分

**同分排序：**
1. 积分 → 2. 净胜场 → 3. 净胜局 → 4. 净胜分

**统计维度：**
- 个人：胜场、败场、净胜场、胜球、输球、净胜球、胜率
- 组合（搭档）：战绩
- 小组（团体）：积分、排名
- 赛种：男双/女双/混双
- 裁判志愿：次数排行 🏅
